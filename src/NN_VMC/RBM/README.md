# FYS4411 Project 2: Neural-Network VMC for Trapped Bosons

This repository contains a Variational Monte Carlo (VMC) code for trapped bosonic systems, extended from the Project 1 code to include a Restricted Boltzmann Machine (RBM) variational wave function. The code can run brute-force Metropolis sampling, importance sampling, parallel replicas and RBM Adam optimization.

## Project structure

```text
.
├── CMakeLists.txt          
├── Makefile                
├── compile_project         # Convenience script to configure and compile the project
├── main.cpp                # Main executable: parses command-line options and launches simulations
├── utilities.cpp/.h        # High-level VMC drivers, parallel replicas, output helpers, optimization utilities
├── system.cpp/.h           # System class connecting Hamiltonian, wave function, solver, and particles
├── particle.cpp/.h         # Particle position storage and update logic
├── rbmoptimizer.h          # Adam optimizer for RBM parameters a, b, and W
├── RBMsampler.cpp/.h       # RBM-specific sampler: energy, gradients, and energy-history accumulation
├── Hamiltonians/           # Elliptic and interacting Hamiltonians
├── WaveFunctions/          # RBM wave function implementation
├── Solvers/                # Metropolis and importance-sampling Monte Carlo solvers
├── InitialStates/          # Initial particle-position generators
├── Math/                   # Random-number utilities 
├── data/
│   ├── .txt/               # Numerical output files from simulations
│   ├── .pdf/               # Generated plots used in analysis/reporting
│   └── .py/                # Python scripts for plotting and statistical analysis
└── build/                  # Build directory generated by CMake
```

## Building

From the project root: /RBM

```bash
make clean
./compile_project
```

The executable is called:

```bash
./vmc
```

## Basic usage

The first argument selects the sampling method:

```bash
./vmc bf <stepLength> [N D] [options]
./vmc is <dt> [N D] [options]
```

where:

- `bf` uses brute-force Metropolis sampling.
- `is` uses importance sampling.
- `N` is the number of particles.
- `D` is the number of spatial dimensions.

Examples:

```bash
./vmc bf 0.8 2 2 --rbm
./vmc is 0.02 2 2 --rbm
./vmc is 0.02 10 3 --interact --rbm
```

## Main options

```text
--opt rbmadam         Optimize RBM parameters using Adam before production.
--lr <value>          Learning rate for RBM Adam optimization.
--Nh <value>          Number of RBM hidden nodes.
--replicas <value>    Number of independent replicas for the production run.
--interact            Include the interaction term in the Hamiltonian.
--gamma <value>       Trap anisotropy parameter for the elliptic trap.
```

Example with RBM optimization and four replicas:

```bash
export OMP_NUM_THREADS=4
./vmc is 0.02 10 3 --interact --opt rbmadam --lr 0.005 --Nh 8 --replicas 4 --gamma 2.82843
```

## Output files

Simulation outputs are written to `data/.txt/`.

Typical output files are:

```text
energy_*.txt          Final energy, acceptance ratio, CPU time, and replica statistics
optimization_*.txt    Energy and acceptance ratio during RBM Adam optimization
gradients_*.txt       Final averaged RBM gradients for a, b, and W
e_history_*.txt       Stored local-energy histories for blocking analysis
```

Plots generated from the Python scripts are stored in `data/.pdf/`.

## RBM workflow

For RBM simulations, the code initializes the RBM parameters

```text
a : N x D
b : Nh
W : N x D x Nh
```

after the command-line values of `N`, `D`, and `Nh` are known. During optimization, the code estimates the VMC energy gradients with respect to `a`, `b`, and `W`, and updates them with Adam. After optimization, the production run uses the optimized parameters and, if requested, averages independent replicas.

## Statistical analysis 

The Python script `data/.py/statistical_analysis.py` is used to analyze the output energy history files `e_history_*.txt` and compute a blocking analysis. The blocking-analysis script can be run from the project root /RBM as (example):

```bash
python3 data/.py/statistical_analysis.py data/.py/elliptical/e_history_is_dt_0p02_RBM_hidden_4_gamma_2p82843_N10_D3.txt
```
## Optimization curves

Optimization curves can be plotted from the `optimization_*.txt` files using the plotting script `data/.py/plot_optimization_comparison`. Generated figures are saved as PDF files in `data/.pdf/`. The plotting script can be run from the project root /RBM as (example):

```bash
python3 data/.py/plot_optimization_comparison.py --N 10 --D 3
```

## Notes

- For large interacting systems, the RBM optimization can be sensitive to the learning rate, number of hidden nodes, and Monte Carlo time step.
- N=100 particles, D=3 and interactions can take long running times.
