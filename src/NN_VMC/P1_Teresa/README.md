# Project 1 – Variational Monte Carlo for trapped bosons

This repository contains the code, analysis scripts, and report material for **Project 1** of Computational Physics II. The project starts from the **non-interacting harmonic oscillator** and progressively extends the VMC framework to brute-force Metropolis and importance sampling, analytic and numerical local-energy evaluation, optimization of the variational parameter \(\alpha\) with a **BFGS-type** method, statistical-error analysis using the **blocking method**, OpenMP-based **parallel replica** runs, a **repulsively interacting hard-sphere** Bose gas in an anisotropic trap, and one-body densities **with and without** the Jastrow factor.

The code is written primarily in **C++14**, with **Python** scripts for plotting and statistical post-processing.

### Main architectural idea

The C++ code is modular:

- **Hamiltonians/** defines how the local energy is computed.
- **WaveFunctions/** defines the trial wave function.
- **Solvers/** defines how the Monte Carlo move is proposed and accepted.
- **InitialStates/** defines how particles are initialized.
- **system.cpp** couples Hamiltonian + wave function + solver + particles.
- **sampler.cpp** accumulates observables during the production run.
- **utilities.cpp** provides the high-level workflows used by `main.cpp`:
  - one VMC run,
  - alpha scans,
  - BFGS optimization,
  - parallel replica runs.

This makes it easy to switch between:
- ideal vs interacting Hamiltonians,
- isotropic vs anisotropic wave functions,
- brute force vs importance sampling,
- normal energy runs vs density-only runs.

## General workflow

A typical workflow is:

1. **Build** the executable.
2. **Run an alpha scan** or **run BFGS optimization** to estimate the best \(\alpha\).
3. **Run a production simulation** using the chosen \(\alpha\).
4. Optionally store:
   - energy histories for blocking analysis,
   - one-body density histograms,
   - optimizer paths,
   - per-replica results.
5. Use the Python scripts in `data/.py/` to:
   - generate plots,
   - perform blocking analysis,
   - compare methods.

## How `main.cpp` works

The executable is `./vmc`.

### Core syntax

There are two main Monte Carlo modes:

- **Brute force Metropolis**
- **Importance sampling**

#### Brute force

```bash
./vmc bf [an|nu] [N D] [options]
```

- `bf` = brute-force Metropolis
- `an` = analytic local energy
- `nu` = numerical Laplacian local energy
- `N` = number of particles
- `D` = number of dimensions

If `an|nu` is omitted in brute-force mode, the default is analytic.

Examples:

```bash
./vmc bf an 100 3
./vmc bf nu 100 3
./vmc bf 100 3
```

#### Importance sampling

```bash
./vmc is <dt> [an] [N D] [options]
```

- `is` = importance sampling
- `<dt>` = time step
- only **analytic** local energy is allowed here

Examples:

```bash
./vmc is 0.005 an 100 3
./vmc is 0.02 an 500 3 --alpha 0.5 --replicas 4
```

Numerical Laplacian with importance sampling is intentionally disallowed.

## Alpha selection modes

`main.cpp` supports three ways to choose \(\alpha\):

### 1. Alpha scan (default)

If neither `--alpha` nor `--opt bfgs` is given, the program performs a scan over a fixed alpha range and stores `E_vs_alpha...txt`.

Example:

```bash
./vmc bf an 100 3
./vmc is 0.005 an 100 3
```

### 2. User-provided alpha

Use `--alpha` to skip the scan.

Example:

```bash
./vmc bf an 100 3 --alpha 0.50
./vmc is 0.005 an 100 3 --alpha 0.50
```

### 3. BFGS optimization

Use:

```bash
--opt bfgs
```

Example:

```bash
./vmc bf an 100 3 --opt bfgs
./vmc is 0.005 an 100 3 --opt bfgs
```

This stores the optimizer path in `opt_path...txt`.

## Parallel replica runs

Parallelization is implemented using **OpenMP** and works by launching several **independent Markov chains** with different seeds.

Use:

```bash
export OMP_NUM_THREADS=4
./vmc is 0.02 an 500 3 --alpha 0.5 --replicas 4
```

or in the interacting brute-force case:

```bash
export OMP_NUM_THREADS=4
./vmc bf an 10 3 --interact --a 0.0043 --gamma 2.82843 --beta 2.82843 --alpha 0.48 --replicas 4
```

### What this does

- each replica is an independent full VMC run,
- results are averaged at the end,
- per-replica energies and histories are stored,
- blocking analysis must still be done **per replica**, not by concatenating replicas into one fake chain.

## Interacting hard-sphere case

The interacting extension is enabled with:

```bash
--interact
```

Additional parameters:

- `--a <value>`: hard-sphere diameter in units of \(a_{ho}\)
- `--gamma <value>`: anisotropy parameter in the Hamiltonian
- `--beta <value>`: anisotropy parameter in the wave function

Typical example:

```bash
./vmc bf an 50 3 --interact --a 0.0043 --gamma 2.82843 --beta 2.82843
```

### Important notes

- The current interacting implementation is intended for **3D**.
- The interacting calculations are run in **brute-force Metropolis** mode.
- The interacting wave function is:
  - anisotropic Gaussian one-body factor,
  - multiplied by a hard-sphere **Jastrow factor**.

## Density-only mode

For exercise 1h, a density-only mode was added to avoid unnecessary local-energy calculations during density estimation.

### Flags

- `--density-only`  
  accumulate only one-body densities
- `--nojastrow`  
  use the anisotropic Gaussian **without** the Jastrow factor
- `--bins <n>`  
  number of histogram bins
- `--zmax <value>`  
  range for the \(z\)-density histogram
- `--rhomax <value>`  
  range for the transverse \(\rho=\sqrt{x^2+y^2}\) histogram

### Example: with Jastrow

```bash
./vmc bf an 50 3 --interact --a 0.0043 --gamma 2.82843 --beta 2.82843 --alpha 0.490065 --density-only
```

### Example: without Jastrow

```bash
./vmc bf an 50 3 --interact --a 0.0043 --gamma 2.82843 --beta 2.82843 --alpha 0.490065 --density-only --nojastrow
```

### Example: custom histogram resolution

```bash
./vmc bf an 50 3 --interact --a 0.0043 --gamma 2.82843 --beta 2.82843 --alpha 0.490065 --density-only --bins 250 --zmax 4.0 --rhomax 3.0
```

## What `main.cpp` writes

The executable writes raw results to `data/.txt/`.

### 1. Alpha-scan files

Examples:

- `E_vs_alpha_mode_bf_N100_D3.txt`
- `E_vs_alpha_mode_is_dt_0p005_N100_D3.txt`
- `E_vs_alpha_hs_mode_bf_N50_D3.txt`

Typical columns:

```text
# alpha  energy  cpu_seconds  acceptance
```

These are used by:
- `plot_E_vs_alpha.py`
- `plot_E_vs_alpha_hs.py`

### 2. Final production summaries

Examples:

- `final_energy_mode_bf_eval_an_N100_D3.txt`
- `final_energy_mode_is_dt_0p02_eval_an_N500_D3.txt`
- `final_energy_hs_mode_bf_eval_an_N50_D3.txt`

In replica-enabled runs, the final summary contains averaged values such as:

```text
# N D mode stepParam omega alpha eval replicas energy_mean gradient_mean acceptance_mean energy_std energy_stderr wall_seconds mean_replica_cpu_seconds
```

These files are used for:
- reporting final energies,
- CPU-time plots,
- comparing methods.

### 3. Energy-history files

Examples:

- old single-chain format:
  - `energy_history_mode_is_dt_0p02_eval_an_N500_D3.txt`
- new replica-aware format:
  - `energy_history_hs_mode_bf_eval_an_N50_D3.txt`

Formats:

Old:
```text
# step local_energy
```

New:
```text
# replica step local_energy
```

These are used by:
- `statistical_analysis.py`

### 4. Optimizer-path files

Examples:

- `opt_path_mode_bf_N1_D1.txt`
- `opt_path_mode_is_dt_0p02_N500_D3.txt`
- `opt_path_hs_mode_bf_N50_D3.txt`

Format:

```text
# iter alpha energy gradient H
```

These are used by:
- `plot_optimizer_path.py`

### 5. Density files

Examples:

- `density_z_hs_withJ_mode_bf_N50_D3.txt`
- `density_z_hs_noJ_mode_bf_N50_D3.txt`
- `density_rho_hs_withJ_mode_bf_N50_D3.txt`
- `density_rho_hs_noJ_mode_bf_N50_D3.txt`

Formats:

```text
# z density
```

or

```text
# rho density
```

These are used by:
- `plot_onebody_densities.py`

### `statistical_analysis.py`

Performs **blocking analysis** of energy histories.

It supports both:
- old single-chain history files,
- new replica-tagged history files.

Run:

```bash
python3 data/.py/statistical_analysis.py data/.txt/energy_history_mode_is_dt_0p02_eval_an_N500_D3.txt
```

or for a replica-tagged file:

```bash
python3 data/.py/statistical_analysis.py data/.txt/energy_history_hs_mode_bf_eval_an_N50_D3.txt
```

What it does:
- reads the energy history,
- splits by replica if needed,
- computes:
  - naive standard error,
  - lag-1 autocorrelation,
  - Flyvbjerg–Petersen blocking estimates,
- writes one report per replica,
- writes a summary file.

Typical outputs:
- `..._replica0_blocking_report.txt`
- `..._blocking_summary.txt`

Dependencies:
- `numpy`
- `argparse`
- `pathlib`

## Suggested workflows

### A. Non-interacting benchmark

1. Run scan:
   ```bash
   ./vmc bf an 100 3
   ```
2. Plot:
   ```bash
   python3 data/.py/plot_E_vs_alpha.py --N 100 --D 3
   ```
3. Run BFGS if desired:
   ```bash
   ./vmc bf an 100 3 --opt bfgs
   ```
4. Run final production:
   ```bash
   ./vmc bf an 100 3 --alpha 0.5
   ```
5. Run blocking analysis on the history file:
   ```bash
   python3 data/.py/statistical_analysis.py data/.txt/energy_history_mode_bf_eval_an_N100_D3.txt
   ```

### B. Parallel replica production

1. Set thread count:
   ```bash
   export OMP_NUM_THREADS=4
   ```
2. Run:
   ```bash
   ./vmc is 0.02 an 500 3 --alpha 0.5 --replicas 4
   ```
3. Analyze:
   ```bash
   python3 data/.py/statistical_analysis.py data/.txt/energy_history_mode_is_dt_0p02_eval_an_N500_D3.txt
   ```

### C. Interacting hard-sphere scan and optimization

1. Run scan:
   ```bash
   ./vmc bf an 50 3 --interact --a 0.0043 --gamma 2.82843 --beta 2.82843
   ```
2. Plot:
   ```bash
   python3 data/.py/plot_E_vs_alpha_hs.py 50
   ```
3. Run BFGS:
   ```bash
   ./vmc bf an 50 3 --interact --a 0.0043 --gamma 2.82843 --beta 2.82843 --opt bfgs
   ```
4. Run final production at the chosen alpha:
   ```bash
   ./vmc bf an 50 3 --interact --a 0.0043 --gamma 2.82843 --beta 2.82843 --alpha 0.490065
   ```
5. Analyze:
   ```bash
   python3 data/.py/statistical_analysis.py data/.txt/energy_history_hs_mode_bf_eval_an_N50_D3.txt
   ```

### D. One-body densities (exercise 1h)

1. Run with Jastrow:
   ```bash
   ./vmc bf an 50 3 --interact --a 0.0043 --gamma 2.82843 --beta 2.82843 --alpha 0.490065 --density-only
   ```
2. Run without Jastrow:
   ```bash
   ./vmc bf an 50 3 --interact --a 0.0043 --gamma 2.82843 --beta 2.82843 --alpha 0.490065 --density-only --nojastrow
   ```
3. Plot:
   ```bash
   python3 data/.py/plot_onebody_densities.py 50
   ```

## Notes and limitations

- Importance sampling is designed for the non-interacting Gaussian workflow. The interacting hard-sphere case is currently run in **brute-force** mode.
- Numerical Laplacian is intended for the non-interacting benchmark comparisons.
- The blocking-analysis script assumes a power-of-two blocking strategy and truncates to the largest \(2^d\) not exceeding the chain length.
- Some scripts are written for legacy files from earlier stages of the project, so always check that the expected `.txt` files already exist before running them.
